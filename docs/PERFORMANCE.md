# Performance

В библиотеке реализована система бенчмаркинга для оценки производительности.

---

## Основная функция

Для измерения производительности используется:

```c
BenchmarkResult benchmark_expression(const char *infix, int iterations, ErrorInfo *err);
````

---

## Параметры

* `infix` — выражение в инфиксной форме
* `iterations` — количество повторений (для усреднения)
* `err` — структура ошибок

---

## Результат

Функция возвращает структуру:

```c
typedef struct {
    double parse_time_ms;
    double eval_time_ms;
    long operations_count;
    size_t memory_used;
} BenchmarkResult;
```

---

## Метрики

### parse_time_ms

Время разбора выражения (в миллисекундах).

---

### eval_time_ms

Время вычисления выражения (в миллисекундах).

---

### operations_count

Количество операций, выполненных при вычислении.

(В текущей реализации может быть приблизительным.)

---

### memory_used

Оценка использованной памяти.

(В текущей версии может быть упрощённой.)

---

## Метод измерения

* используется `clock_gettime`
* применяется монотонный таймер (`CLOCK_MONOTONIC`)
* выполняется несколько итераций
* результат суммируется

---

## Пример использования

```c
ErrorInfo err;

BenchmarkResult res = benchmark_expression("3 + 4 * 2", 1000, &err);

if (err.code == ERR_OK) {
    printf("Parse time: %f ms\n", res.parse_time_ms);
    printf("Eval time : %f ms\n", res.eval_time_ms);
}
```

---

## Интерпретация результатов

* меньшее значение времени → лучше производительность
* большое число операций → более сложное выражение
* увеличение времени при росте выражения — нормальное поведение

---

## Ограничения

* `memory_used` может быть неточным
* `operations_count` может быть приближённым
* результаты зависят от системы и нагрузки CPU
* не учитываются кэш и оптимизации компилятора

---

## Рекомендации

* использовать не менее 1000 итераций
* тестировать разные типы выражений:

  * простые (1+2)
  * сложные (вложенные скобки)
  * с функциями
  * с переменными
